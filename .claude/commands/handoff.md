## 제약

- **50줄 상한**. 초과 시 덜 중요한 항목 버리거나 memory 로 승격.
- **3섹션 이외 추가 금지**. 자유 서식은 누적 문제를 다시 부른다.

## 넣지 말 것 (다른 레이어에 이미 있음 → 중복 낭비)

- 코드 변경 사항 → `git log` 로 확인 가능
- 디자인·코딩 규칙 → `prototypes/README.md`, memory (`MEMORY.md` 인덱스)
- 과거 대화 맥락 → 이전 세션 컨텍스트 재현은 의도적으로 하지 않음

## 판단 기준

"다음 세션이 코드·git·memory·plan 파일만으로는 재구성할 수 없는 이행적 상태인가?" → 맞으면 넣고, 아니면 뺀다.

"무엇을 했다"보다 **"다음 세션이 이 맥락을 알아야 올바른 판단을 할 수 있다"** 만 남긴다.
