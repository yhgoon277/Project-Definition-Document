## §1 Project Context

SKT Next Channel 리뉴얼의 일환으로, **"고객 주문경험 혁신 프로세스 재설계"** 과제를 Toss 스타일 모바일 앱으로 재설계한 High-fidelity 인터랙티브 프로토타입을 만듭니다.

**과제 목적**
- 표준화된 공통 주문 프로세스 수립으로 채널·상품 무관 일관된 경험 제공
- 모듈형·재사용 가능한 설계로 확장성 확보

**타깃 사용자**: SKT 모바일 요금제·단말을 앱에서 직접 가입하려는 20~50대 개인고객.

**핵심 컨셉(전략가치)**: 셀프완결 · 원스톱 통합

**현재(AS-IS) 문제**
- 상품/유형별 주문 단계와 인증 방식이 제각각
- 앱/웹 이원화로 복잡성 증가, 흐름 중간에 채널 이동 발생
- 비대면 완결률 저조 → 오프라인 의존
- 복잡한 단계·반복 입력·채널별 다른 흐름

**지향점(TO-BE)**
- **표준 모듈 구조**: 가입·인증·약관·결제·개통을 공통 모듈로 분리, 재조합 가능
- **입력량 최소화**: "생각할 양"과 "입력할 양"을 줄임
- **고객 정보 재활용**: 보유 정보·직전 입력값·인증 결과 자동 반영, 이탈 후 복원
- **막힘 순간 즉시 이해**: 실패 원인·영향·다음 행동을 진행 중 바로 제공
- **AI 주문 도우미**: 단계별 시각 안내, 개통 가이드, 상담 연결까지 AI 지원
- **셀프개통을 독립 여정으로**: 단계형 UX, 오류 복구, 도움 연결
- **주문의 끝 = 실사용 시작**: 개통→혜택 적용→서비스 연동까지 하나의 흐름

## §2 Design System (Toss-inspired Tokens)

아래 토큰을 **모든 스크린에 일관 적용**. Toss의 시그니처 패턴(스텝형 전체화면, 큰 숫자, 한 화면 한 결정, 진행률 상단 바)을 참고합니다.

```
Color
  primary       : #3182F6
  primary-weak  : #4E91F7
  text-primary  : #191F28
  text-secondary: #4E5968
  text-tertiary : #8B95A1
  bg            : #FFFFFF
  surface       : #F2F4F6
  success       : #22C55E
  danger        : #F04438
  warn          : #F79009
  info-bg       : #E8F3FF

Typography (Pretendard)
  H1       28 / 36  Bold
  H2       22 / 28  Bold
  Title    18 / 24  Semibold
  Body     15 / 22  Regular
  Caption  13 / 18  Regular
  Amount   32 / 40  Bold (금액·월 납부액 전용)

Spacing (8pt): 4, 8, 12, 16, 20, 24, 32, 48
Radius: card 16, button 12, chip 999
Shadow: card 0 1px 3px rgba(0,0,0,.04)

Signature patterns
  - 스텝 네비: 상단 4px 진행률 바(primary), 좌측 백버튼, 우측 "이어하기 저장"
  - 전체화면 스텝: 한 화면 = 한 질문/선택. 하단 풀블리드 Primary CTA 56px
  - 선택 카드: radius 16, 선택 시 primary-weak 배경 + primary 1.5px 보더
  - 자동 채워진 필드: 우측에 작은 "자동 입력됨" 칩(primary-weak 배경)
  - 오류: 필드 하단에 danger 캡션 + 해결 가이드 링크(primary 텍스트)
```

**글로벌 흐름**: 홈 → 상품 둘러보기(비로그인 허용) → 주문 시도(로그인) → 정보 입력 → 약관·인증 → 결제 → 개통 → 온보딩.

## §3 Screen List (네비게이션 연결 포함)

총 **10개 화면**을 feature 4개로 구성. #8의 features 7개 중 painPoints와 직결된 4개만 선정.

### Feature 1. 상품 선택/구성 모듈

**Screen 1 [AS-IS] — 분산된 상품 탐색**
- 구성: 상단 탭 "단말 / 요금제 / 결합할인 / 사은품"이 서로 다른 페이지로 이동. 단말 선택 후 요금제 페이지로 넘어가면 직전 선택값이 사라짐. 예상 금액은 "주문하기" 단계에서만 확인 가능. 실제 T world / T-다이렉트 구조 재현.
- 문제 표시: "단계 분절 · 비교·시뮬레이션·추천 미연결 · 중도 이탈 시 정보 소실"

**Screen 2 [TO-BE] — 통합 상품 구성 스텝**
- 구성:
  - 상단 진행률 바 (1/4)
  - H1 "어떤 폰으로 바꿀까요?"
  - 단말 선택 카드 리스트 (이미지 + 모델명 + "월 42,000원부터")
  - 한 장 선택 시 즉시 아래에 요금제 섹션이 펼쳐짐(아코디언 스텝 2/4)
  - 하단 **실시간 예상 금액 고정 바**: "월 예상 62,400원 · 자세히 →"
  - 하단 CTA: "다음"
- Nav: CTA → Screen 3(AI 추천 시뮬레이션). "자세히" → 바텀시트 상세 breakdown.

**Screen 3 [TO-BE] — AI 추천 시뮬레이션**
- 구성:
  - H2 "제 사용 패턴에 맞는 조합이에요"
  - AI 배지 + "최근 6개월 사용량 기준 추천"
  - 비교 카드 3장: [내가 고른 조합] / [AI 추천 ①] / [AI 추천 ②]
  - 각 카드에 월 금액(Amount), 주요 차이 3줄, "이걸로 바꾸기" 링크
  - 하단 CTA: "이대로 주문하기" (Primary)
- Nav: CTA → Screen 4(AS-IS 비교용). "이걸로 바꾸기" → Screen 2로 돌아가 선택 반영.

### Feature 2. 주문 정보 입력 표준화

**Screen 4 [AS-IS] — 빈 폼 반복 입력**
- 구성: "배송지 주소" / "청구지 주소" / "연락처" / "생년월일" / "본인 명의 여부" 모든 필드가 빈 폼. 명의 인증 후 다시 배송지에서 동일 정보 요구. 주문 도중 앱 종료 시 전부 초기화.
- 문제 표시: "반복 입력 · 이탈 후 복원 불가 · 재입력 피로"

**Screen 5 [TO-BE] — 자동 입력 + 이어하기**
- 구성:
  - 상단 "주문 정보 확인" + 진행률 2/4
  - 카드 "배송지 · 서울시 중구 을지로 XX · 자동 입력됨" (우상단 "자동 입력됨" 칩)
  - "다른 곳으로 받을게요" Secondary 버튼
  - 카드 "청구지 · 배송지와 동일" (토글)
  - 카드 "본인 인증 · PASS 인증 완료 (2일 전)" (이전 인증 재활용 배지)
  - 상단 고정 알림: "주문 도중 앱을 닫아도 언제든 이어서 할 수 있어요" (info-bg)
  - 하단 CTA: "다음"
- Nav: CTA → Screen 6.

### Feature 3. 결제·셀프개통 연속 흐름

**Screen 6 [AS-IS] — 결제 완료·개통 단절**
- 구성: "주문이 완료되었습니다. 주문번호 20260420-XXX". 개통 안내 없음. 배송 도착 후 "셀프 개통은 앱의 개통 메뉴에서..."만 안내. 실패 시 "오류 코드 E-203 · 고객센터 문의"만 표시.
- 문제 표시: "개통이 별도 여정 · 오류 시 셀프 해결 불가 · CS 의존"

**Screen 7 [TO-BE] — 단계형 셀프개통 가이드**
- 구성:
  - 상단 "박스를 받으셨나요?" H1
  - 3단계 스테퍼: ① 유심 꺼내기 → ② 폰에 끼우기 → ③ 앱에서 개통
  - 각 단계 일러스트 카드(둥근 배경 #F2F4F6)
  - 현재 단계 "유심 꺼내기" 진행 중 상태 강조
  - 하단 CTA: "유심 꺼냈어요, 다음"
- Nav: CTA → Screen 8.

**Screen 8 [TO-BE] — AI 개통 도우미 (오류 복구)**
- 구성:
  - 에러 상태 가정: "유심이 인식되지 않아요" (danger 타이포)
  - AI 말풍선: "다음 중 어떤 상황이에요?" 선택지 3개
    - "유심을 뒤집어 끼운 것 같아요"
    - "폰이 재부팅 중이에요"
    - "잘 모르겠어요"
  - "잘 모르겠어요" 선택 시 → 실시간 상담 연결 배너 노출
  - 하단 고정 "다시 시도하기" (Primary) / "상담 연결" (Secondary Text)
- Nav: "다시 시도하기" → 성공 시 Screen 9.

### Feature 4. 온보딩 — 주문의 끝 = 실사용 시작

**Screen 9 [AS-IS] — 가입 완료 후 무안내**
- 구성: "개통이 완료되었습니다" 단일 문구 + 홈으로 가기 버튼. 혜택/멤버십/구독/결합할인 자동 적용 여부 알 수 없음. 이후 할 일도 미제공.
- 문제 표시: "가입 후 즉시 사용 불가 · 혜택 적용 여부 불투명"

**Screen 10 [TO-BE] — 실사용 시작 온보딩**
- 구성:
  - 성공 애니메이션(파란 체크 확장) + H1 "축하해요, 이제 쓸 수 있어요"
  - 자동 연결 체크리스트(모두 ✅):
    - "T멤버십 VIP 자동 승계"
    - "결합할인(가족 4회선) 자동 적용 · 월 -12,000원"
    - "FLO·Wavve 구독 혜택 연결"
    - "부모님 회선 요금 합산 확인"
  - **다음 할 일 카드**:
    - "첫 달 예상 요금 보기 →" (#4 요금 허브 링크)
    - "멤버십 혜택 둘러보기 →"
    - "eSIM 세컨폰 추가하기 →"
  - 하단 CTA: "요금 허브로 가기" (Primary)
- Nav: CTA → 요금 허브(#4 프로토타입으로 연결 placeholder).

## §4 전역 인터랙션/상태

- 모든 스텝 전환은 좌→우 슬라이드(220ms, cubic-bezier .4,0,.2,1)
- 진행률 바는 단계 변경 시 부드럽게 width 애니메이션
- "자동 입력됨" 칩은 hover/tap 시 "이 정보는 어디서 왔어요?" 바텀시트(출처·수정 경로)
- AS-IS 화면 우상단에 "현재(AS-IS)" 배지, TO-BE에 "개선안(TO-BE)" 배지
- 오류 상태는 필드만 danger 테두리, 전체 화면 톤은 유지(패닉 유도 금지)

## §5 산출물 요구사항

1. 10개 화면이 **하나의 주문 여정**으로 연결되어, 상품 선택(Screen 2)부터 온보딩(Screen 10)까지 한 흐름으로 탐색 가능해야 합니다.
2. AS-IS 화면(1,4,6,9)은 TO-BE 화면으로 이어지는 지점에 "→ 개선안 보기" 스위치 토글을 배치해 1:1 대비가 명확하게.
3. 결과는 공유 URL, PPTX, Claude Code 핸드오프 번들로 내보낼 수 있게 준비.
