# 과제정의서 발췌 — #4 신뢰 기반 셀프완결형 요금 납부·관리 경험 재설계

(Claude Design 프로젝트에 첨부하는 상세 자료)

- 문서번호: TK_CH_BL
- 고객여정: 채널 운영/경험
- 업무영역: 요금 조회/납부
- 카테고리: 요금/결제 (billing)

## 목적

1. 청구·납부 정보를 고객 관점에서 재해석하고 의미 이해 중심 구조로 전환
2. 사후 확인에서 사전 예측·관리 가능한 구조로 전환
3. 자동납부·납부수단·실패 가능성까지 포함한 사전 관리 경험 강화
4. 조회-납부-변경-예외 대응 간 셀프완결형 처리 기반 구축
5. 개인 중심을 넘어 가족·다회선·위임 처리까지 확장

## 현재 상태 (AS-IS)

- 기능 분산 구조로 여정 단절
- 청구 금액 중심 나열
- 생활 단위 시나리오 미흡
- 예외 상황 대응 한계

## Pain Points

- 금액 이해 곤란 — 왜 이 금액인지 설명 부족
- 다음 달 예측 어려움
- 미납·자동납부 실패 해결 곤란
- 메뉴 탐색 부담
- 권한 장벽 (가족·위임)

## 근본 원인

- IA와 처리 구조 분절
- 고객 언어 부재 (시스템 용어 그대로 노출)
- 정보 통합 부재
- 정책 디지털 완결성 부족

## 핵심 기능 (파일럿에서 모두 커버)

1. 상태 파악 및 요금/납부 통합 허브
2. 현재 요금 및 미납 상태 확인
3. 자동납부 정책 관리
4. 납부수단 변경

## 지향점 및 기대효과

| 지향점 | 설명 | 기대효과 |
|---|---|---|
| 청구 언어 재구성 | 시스템 용어 → 고객 언어, "왜 이 금액인지" 의미 중심 설명 | 청구/과금 불신 해소, 요금 VOC 감소 |
| 납부 상태·리스크 관리 | 미납·자동납부 실패 사전 안내 + 즉시 해결 경로 | 미납 장기화 방지, 결제 실패 CS 감소 |
| 예측형 요금 관리 | 다음 달 예상 금액, 변경 시 영향 사전 시뮬레이션 | 예측 가능한 요금 경험, 불안감 해소 |
| 원스톱 해결 흐름 | 조회→납부→변경→예외 대응을 하나의 흐름으로 연결 | 셀프완결 처리율 향상 |
| 가족·다회선 확장 | 가족·위임·다회선 요금 관리 통합 | 권한 장벽 해소, 가족 단위 만족도 향상 |
| 24시간 신뢰 기반 채널 | 시간 무관 요금 조회·납부·변경 셀프 처리 | 야간/주말 긴급 문의 감소 |

## 주요 프로세스 및 기능 (전 여정)

### 1. 상태 인식
요금/납부 통합 허브 진입 · 현재 요금 상태 확인 · 미납/자동납부 상태 확인 · 납부 수단 상태 확인 · 가족/다회선 요금 요약 · 알림/리마인드

### 2. 청구 이해
이번 달 청구 항목별 설명 · 전월 대비 변동 안내 · 부가서비스·할인 항목 확인 · 소액결제/콘텐츠 이용 내역 · 세금/공제 항목 설명

### 3. 예측·판단
다음 달 예상 금액 시뮬레이션 · 요금제 변경 시 영향 미리보기 · 할인·프로모션 적용 시뮬레이션 · 약정 종료 시 변화 안내 · 최적 요금제 추천

### 4. 실행
즉시 납부 · 부분 납부 · 납부 수단 변경 · 자동납부 설정/변경 · 납부 예약 · 영수증 발급

### 5. 조건 변경
납부일 변경 · 청구서 수신 방식 변경 · 청구 통합/분리 · 위임 납부 설정

### 6. 예외 해결
미납 해결 가이드 · 자동납부 실패 복구 · 이중 납부 확인/환불 · 분쟁 접수 및 처리 상태 조회 · 상담 연결

### 7. 완료·후속
납부 완료 확인 · 변경 적용 결과 안내 · 다음 달 예상 안내 · 관련 업무 추천

- #3 계약 및 가입 정보 관리 체계 — 회선/명의 권한 연동
