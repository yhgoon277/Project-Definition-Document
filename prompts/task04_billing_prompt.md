## §1 Project Context

SKT Next Channel 리뉴얼의 일환으로, **"신뢰 기반 셀프완결형 요금 납부·관리 경험 재설계"** 과제를 Toss 스타일 모바일 앱으로 재설계한 High-fidelity 인터랙티브 프로토타입을 만듭니다.

**과제 목적**
- 청구·납부 정보를 고객 관점에서 재해석하고 의미 이해 중심 구조로 전환
- 사후 확인 → 사전 예측·관리 가능한 구조로 전환
- 자동납부·납부수단·실패 가능성까지 포함한 사전 관리 경험 강화
- 조회–납부–변경–예외 대응을 셀프완결형으로 연결
- 개인 중심 → 가족·다회선·위임 처리까지 확장

**타깃 사용자**: SKT 요금을 앱으로 처리하는 20~60대 개인고객 및 가족 대표자(가장).

**핵심 컨셉(전략가치)**: 셀프완결 · 신뢰 기반 설계

**현재(AS-IS) 문제**
- 기능 분산 구조로 요금 여정이 단절
- 청구 금액이 시스템 용어와 숫자로만 나열, "왜 이 금액인지" 설명 부족
- 다음 달 요금 예측 어려움
- 미납·자동납부 실패 시 해결 경로 불명확 → CS 의존
- 가족·위임 처리 시 권한 장벽

**지향점(TO-BE)**
- **청구 언어 재구성**: 시스템 용어를 고객 언어로, "왜 이 금액인지" 의미 중심 설명
- **납부 상태·리스크 관리**: 미납·자동납부 실패를 사전 안내 + 즉시 해결 경로
- **예측형 요금 관리**: 다음 달 예상 금액과 요금제 변경 영향 시뮬레이션
- **원스톱 해결 흐름**: 조회→납부→변경→예외 대응을 하나의 흐름으로
- **가족·다회선 확장**: 가족·위임·다회선 요금을 통합 관리

## §2 Design System (Toss-inspired Tokens)

아래 토큰을 **모든 스크린에 일관 적용**하세요. Toss 앱의 실제 패턴(홈 카드 스택, 리스트 로우 2줄, 바텀시트, 전체화면 스텝)을 참고합니다.

```
Color
  primary       : #3182F6  (Toss Blue — 주요 CTA·링크·포커스)
  primary-weak  : #4E91F7
  text-primary  : #191F28  (본문/숫자)
  text-secondary: #4E5968  (보조 설명)
  text-tertiary : #8B95A1  (캡션/미세)
  bg            : #FFFFFF
  surface       : #F2F4F6  (카드 배경, 구분)
  success       : #22C55E
  danger        : #F04438  (미납·경고 타이포)
  warn          : #F79009
  info-bg       : #E8F3FF  (Blue weak surface)

Typography (Pretendard, -apple-system fallback)
  H1       28 / 36  Bold    — 화면 제목(금액 강조 시)
  H2       22 / 28  Bold    — 섹션 제목
  Title    18 / 24  Semibold— 카드 제목
  Body     15 / 22  Regular — 본문
  Caption  13 / 18  Regular — 보조/주석
  Amount   32 / 40  Bold    — 금액 전용 (원 단위는 18px 작게, 뒤 여백 2pt)

Spacing (8pt grid): 4, 8, 12, 16, 20, 24, 32, 48
Radius          : card 16, button 12, chip 999, bottomsheet 20 top-only
Shadow          : card 0 1px 3px rgba(0,0,0,.04), floating 0 8px 24px rgba(0,0,0,.08)

Signature patterns
  - 금액 표현: "1,234,567" + 작은 "원" 단위 (Amount 토큰)
  - Primary CTA: 화면 하단 풀블리드, 56px 높이, radius 12, text-white 17/24 Semibold
  - 한 화면 = 한 결정 (스텝 구조). 입력·선택은 바텀시트나 전체화면 스텝으로 분리
  - 상단 네비: 좌측 Chevron-left 백버튼, 타이틀 좌정렬(중앙 타이틀 금지)
  - 상태 배지: primary-weak 배경 + primary 텍스트, 위험은 danger 타이포만(배경 사용 X)
  - 긍정 감정은 파란색 계열, 빨강은 "미납·실패" 경고에만 국한
```

**하단 탭바(글로벌 네비)**: 홈 / 요금·납부(현재 허브) / MY

## §3 Screen List (네비게이션 연결 포함)

총 **10개 화면**을 feature 4개로 구성합니다. AS-IS 화면은 현재 SKT T world의 실제 분산 구조를 재현, TO-BE는 위 지향점을 반영한 개선판.

### Feature 1. 요금·납부 통합 허브 진입

**Screen 1 [AS-IS] — 분산 메뉴 페이지**
- 구성: 상단 탭 "요금 조회 / 납부 / 자동납부 / 영수증 / 사용량" 5개가 별도 메뉴 링크로 나열. 메뉴 아래는 단순 설명 텍스트. 가족·다회선은 별도 메뉴 깊이 2단계 아래.
- 문제 표시(주석): "기능 분산 구조로 여정 단절 · 메뉴 탐색 부담"
- Nav: 백버튼 → 홈. 각 메뉴 탭 → 별도 페이지(데드엔드).

**Screen 2 [TO-BE] — 요금·납부 허브**
- 구성:
  - 상단 인사 "OO님의 이번 달 요금"
  - **이번 달 청구 카드**: Amount 타이포로 "68,200원", 전월 대비 "+3,200원 ▲", "7월 25일 자동납부 예정" 캡션
  - **미납/자동납부 리스크 배너**(해당 시): "6월분 미납 12,000원 · 지금 해결하기" CTA, danger 타이포
  - **다음 달 예상 카드**: "예상 72,400원", "요금제 변경 시뮬레이션 →" 링크
  - **가족 회선 요약**: 부모/자녀 2명, 각 월납 금액 미니 리스트
  - 하단 고정: "즉시 납부" Primary CTA
- Nav: "이번 달 청구 카드" 탭 → Screen 4 (청구 이해). "리스크 배너" → Screen 7(자동납부). "다음 달 예상" → Screen 5(예측 시뮬레이션). "가족 회선" → 가족 허브(미포함, placeholder).

### Feature 2. 현재 요금 및 미납 상태 확인

**Screen 3 [AS-IS] — 청구서 나열 화면**
- 구성: T-Direct / T-World 전형 청구서. "기본료 55,000 / 부가서비스 3,300 / 할인 -2,100 / 세금 1,000" 같은 항목이 표 형태로 나열. 전월 대비 정보 없음. 고객 이해 어려운 내부 용어 그대로.
- 문제 표시: "시스템 용어 그대로 노출 · 왜 이 금액인지 설명 부족"

**Screen 4 [TO-BE] — 청구 의미 재구성**
- 구성:
  - 상단 Amount "68,200원" + 캡션 "이번 달 청구"
  - **"왜 이 금액이에요?"** 섹션: 카드 3장(스택)
    1. "기본요금 55,000원 · 5G 슬림+ 요금제"
    2. "추가 사용 +3,200원 · 데이터 초과 2.3GB" (아이콘: 데이터)
    3. "할인 -2,100원 · T멤버십 VIP 할인 적용"
  - **전월 대비** 카드: "지난달보다 +3,200원, 주로 데이터 초과 때문"
  - **비슷한 달을 살펴봤어요** 인사이트: "지난 3개월 평균 65,000원"
  - 하단 CTA: "다음 달 예상 시뮬레이션" (Secondary)
- Nav: 각 카드 탭 → 바텀시트 상세. 하단 CTA → Screen 5.

**Screen 5 [TO-BE] — 예측 시뮬레이션**
- 구성:
  - Amount "예상 72,400원" (다음 달)
  - 슬라이더/옵션: "요금제 변경 시 얼마?" — 현재 요금제 vs 추천 요금제 비교 리스트
  - 변경 시 "첫 달 영향 +4,000원 · 두 달째부터 -8,000원" 안내
  - "내게 맞는 요금제 추천" AI 배지
  - 하단 CTA: "요금제 바꾸기" (Primary)
- Nav: CTA → 별도 플로우(placeholder). 백버튼 → Screen 4.

### Feature 3. 자동납부 정책 관리

**Screen 6 [AS-IS] — 자동납부 설정 단순 ON/OFF**
- 구성: 토글 "자동납부 ON" / 결제수단 "신한카드 ****1234". 실패 이력·리스크 정보 없음. 변경하려면 인증 후 새 카드 재등록.
- 문제 표시: "실패 가능성·복구 경로 미제공 · 권한 장벽"

**Screen 7 [TO-BE] — 자동납부 상태·리스크 허브**
- 구성:
  - 상단 카드: "자동납부 활성화 · 다음 출금 7/25"
  - **리스크 알림**: "카드 유효기간 9월 만료 예정 · 지금 바꾸기" (danger 타이포만)
  - **최근 결제 이력**: 월별 성공/실패 칩 리스트 (최근 6개월)
  - **실패 복구 가이드** 섹션(과거 실패 있을 시): "5월 실패 → 잔액 부족 → 3일 후 자동 재시도 성공" 타임라인
  - 하단 CTA 2개: "결제수단 바꾸기" Primary / "자동납부 끄기" Text
- Nav: "결제수단 바꾸기" → Screen 9. "카드 유효기간" 알림 → 결제수단 변경 바텀시트 바로 열림.

### Feature 4. 납부수단 변경

**Screen 8 [AS-IS] — 결제수단 등록 화면**
- 구성: 카드번호·유효기간·CVC·생년월일 입력 폼. 기존 수단 제거 → 신규 등록 순서. 변경 적용 시점 불명. 실패 시 "오류 코드 E-102"만 노출.
- 문제 표시: "오류 원인 불명 · 적용 시점 불투명"

**Screen 9 [TO-BE] — 영향 미리보기 + 변경 적용**
- 구성:
  - 상단: 현재 수단 카드(신한 1234) → 새 수단 카드(KB국민 5678) 비교 뷰
  - **적용 일정 카드**: "7/22 00시부터 적용. 7/25 자동납부는 새 수단으로 진행됩니다."
  - **혜택 변화**: "KB 제휴 5% 할인 자동 적용, 월 최대 2,000원"
  - **되돌리기 안내**: "변경 후 24시간 내 취소 가능"
  - 하단 CTA: "KB국민카드로 바꾸기" (Primary)
- Nav: CTA → Screen 10 완료.

**Screen 10 [TO-BE] — 변경 완료·후속 안내**
- 구성:
  - 성공 아이콘(파란 체크) + "변경이 완료되었어요"
  - 요약 카드: "다음 자동납부부터 KB국민카드로 출금됩니다"
  - **다음 할 일** 리스트: "다음 달 예상 금액 확인하기 →" / "가족 회선도 같이 관리하기 →"
  - 하단 CTA: "요금 허브로 돌아가기"
- Nav: CTA → Screen 2(허브).

## §4 전역 인터랙션/상태

- 모든 금액 변동은 숫자 카운트업 애니메이션(300ms)
- AS-IS 화면 우상단에 "현재(AS-IS)" 배지, TO-BE 화면 우상단에 "개선안(TO-BE)" 배지를 작게 붙여 비교 의도를 명시
- 바텀시트는 상단 grabber(4×36 rounded), 배경 dim 60%
- 로딩: Toss식 스켈레톤(회색 블록), 스피너 지양
- 성공 피드백: Primary 컬러 체크 + 가벼운 바이브 피드백 연상되는 마이크로 모션(0.2s scale 1→1.05→1)
