## §1 Project Context

SKT Next Channel 리뉴얼의 일환으로, **"신뢰 기반 셀프완결형 요금 납부·관리 경험 재설계"** 과제를 모바일 앱으로 재설계한 High-fidelity 인터랙티브 프로토타입을 만듭니다.

**과제 목적**
- 청구·납부 정보를 고객 관점에서 재해석하고 의미 이해 중심 구조로 전환
- 사후 확인 → 사전 예측·관리 가능한 구조로 전환
- 자동납부·납부수단·실패 가능성까지 포함한 사전 관리 경험 강화
- 조회–납부–변경–예외 대응을 셀프완결형으로 연결
- 개인 중심 → 가족·다회선·위임 처리까지 확장

**타깃 사용자**: SKT 요금을 앱으로 처리하는 20~60대 개인고객 및 가족 대표자(가장).

**핵심 컨셉(전략가치)**: 셀프완결 · 신뢰 기반 설계

**현재(AS-IS) 문제**
- 기능 분산 구조로 요금 여정이 단절
- 청구 금액이 시스템 용어와 숫자로만 나열, "왜 이 금액인지" 설명 부족
- 다음 달 요금 예측 어려움
- 미납·자동납부 실패 시 해결 경로 불명확 → CS 의존
- 가족·위임 처리 시 권한 장벽

**지향점(TO-BE) 핵심 원칙**
- **청구 언어 재구성**: 시스템 용어를 고객 언어로, "왜 이 금액인지" 의미 중심 설명
- **납부 상태·리스크 관리**: 미납·자동납부 실패를 사전 안내 + 즉시 해결 경로
- **예측형 요금 관리**: 다음 달 예상 금액과 요금제 변경 영향 시뮬레이션
- **원스톱 해결 흐름**: 조회→납부→변경→예외 대응을 하나의 흐름으로
- **가족·다회선 확장**: 가족·위임·다회선 요금을 통합 관리

## §2 정보 구조 & 네비게이션

- **허브형 구조**: 상단에 허브(Screen 2)를 두고 feature별 상세로 진입·복귀
- **하단 탭바 (글로벌 네비)**: 홈 / 요금·납부 / MY
- 모든 스크린은 좌상단 백버튼으로 직전 스크린 복귀 가능해야 하며, Screen 2(허브)에서 출발해 Screen 10(완료)까지 클릭만으로 완주 가능해야 합니다.

## §3 Screen List (10개)

총 **10개 화면**을 4개 feature로 구성합니다. 각 feature는 **AS-IS 1화면 + TO-BE 1~2화면** 쌍으로 묶여 전·후 비교가 명확해야 합니다.

모든 **AS-IS 화면은 우상단에 "현재(AS-IS)" 태그**, **TO-BE 화면은 "개선안(TO-BE)" 태그**를 작은 배지로 표시해 비교 의도를 명시해 주세요.

### Feature 1. 요금·납부 통합 허브 진입

**Screen 1 [AS-IS] — 분산 메뉴 페이지**
- 상단에 "요금 조회 / 납부 / 자동납부 / 영수증 / 사용량" 5개 메뉴가 개별 링크 나열. 본문은 짧은 안내 텍스트만.
- 가족·다회선은 별도 메뉴 깊이 2단계 아래에 존재.
- 각 메뉴 탭 클릭 시 데드엔드 페이지로 분기.
- 화면 하단 주석: "기능 분산 구조로 여정 단절 · 메뉴 탐색 부담"

**Screen 2 [TO-BE] — 요금·납부 허브**
- 상단 인사 "OO님의 이번 달 요금"
- **이번 달 청구 카드**: 금액 "68,200원"을 큰 숫자로 강조, 전월 대비 "+3,200원 ▲" 변동 표기, "7월 25일 자동납부 예정" 보조 캡션
- **미납/리스크 배너**(조건부): "6월분 미납 12,000원 · 지금 해결하기" 즉시 행동 유도
- **다음 달 예상 카드**: "예상 72,400원" + "요금제 변경 시뮬레이션 →" 링크
- **가족 회선 요약**: 부모·자녀 2명, 각 월 납부 금액 미니 리스트
- 하단 고정 CTA: "즉시 납부"
- 네비게이션:
  - 이번 달 청구 카드 → Screen 4
  - 리스크 배너 → Screen 7
  - 다음 달 예상 → Screen 5
  - 가족 회선 → 가족 허브(이번 범위 외, placeholder 화면만)

### Feature 2. 현재 요금 및 미납 상태 확인

**Screen 3 [AS-IS] — 청구서 나열 화면**
- 기존 T world / T-Direct 전형 청구서 형태. "기본료 55,000 / 부가서비스 3,300 / 할인 -2,100 / 세금 1,000" 항목이 표 형태로 나열.
- 전월 대비 정보 없음. 내부 용어(예: "집계 청구 금액", "당월 부가세") 그대로 노출.
- 주석: "시스템 용어 그대로 노출 · 왜 이 금액인지 설명 부족"

**Screen 4 [TO-BE] — 청구 의미 재구성**
- 화면 상단 큰 숫자 "68,200원" + 캡션 "이번 달 청구"
- **"왜 이 금액이에요?"** 섹션, 카드 3장 스택:
  1. "기본요금 55,000원 · 5G 슬림+ 요금제"
  2. "추가 사용 +3,200원 · 데이터 초과 2.3GB" (아이콘: 데이터)
  3. "할인 -2,100원 · T멤버십 VIP 할인 적용"
- **전월 대비** 카드: "지난달보다 +3,200원, 주로 데이터 초과 때문"
- **비슷한 달을 살펴봤어요** 인사이트: "지난 3개월 평균 65,000원"
- 각 카드 탭 시 바텀시트로 상세 풀림
- 하단 CTA(Secondary): "다음 달 예상 시뮬레이션" → Screen 5

**Screen 5 [TO-BE] — 예측 시뮬레이션**
- 큰 숫자 "예상 72,400원" (다음 달)
- **요금제 변경 시 얼마?** 비교 리스트 — 현재 요금제 vs 추천 요금제 2개
- 변경 시 영향 안내: "첫 달 +4,000원 · 두 달째부터 -8,000원"
- AI 배지 + "내게 맞는 요금제 추천"
- 하단 CTA: "요금제 바꾸기" (별도 플로우 placeholder)

### Feature 3. 자동납부 정책 관리

**Screen 6 [AS-IS] — 자동납부 설정 단순 ON/OFF**
- 토글 "자동납부 ON" / 등록 수단 "신한카드 ****1234" 한 줄 표시.
- 실패 이력·리스크 정보 없음. 변경하려면 인증 후 새 카드 재등록.
- 주석: "실패 가능성·복구 경로 미제공 · 권한 장벽"

**Screen 7 [TO-BE] — 자동납부 상태·리스크 허브**
- 상단 카드: "자동납부 활성화 · 다음 출금 7/25"
- **리스크 알림** 카드: "카드 유효기간 9월 만료 예정 · 지금 바꾸기" (경고 톤)
- **최근 결제 이력**: 최근 6개월 성공/실패 칩 리스트 (월별 가로 스크롤 또는 리스트)
- **실패 복구 가이드** 섹션(과거 실패가 있을 경우): "5월 실패 → 잔액 부족 → 3일 후 자동 재시도 성공" 타임라인
- 하단 CTA 2개: "결제수단 바꾸기" (Primary) → Screen 9 / "자동납부 끄기" (Text)
- "카드 유효기간" 알림 클릭 시 결제수단 변경 바텀시트 즉시 열림

### Feature 4. 납부수단 변경

**Screen 8 [AS-IS] — 결제수단 등록 화면**
- 카드번호·유효기간·CVC·생년월일 빈 폼. 기존 수단 제거 → 신규 등록 순서.
- 변경 적용 시점 불명. 실패 시 "오류 코드 E-102"만 노출.
- 주석: "오류 원인 불명 · 적용 시점 불투명"

**Screen 9 [TO-BE] — 영향 미리보기 + 변경 적용**
- 상단에 현재 수단 → 새 수단 비교 뷰 (카드 2장 가로 배치)
- **적용 일정 카드**: "7/22 00시부터 적용. 7/25 자동납부는 새 수단으로 진행됩니다."
- **혜택 변화 카드**: "KB 제휴 5% 할인 자동 적용, 월 최대 2,000원"
- **되돌리기 안내**: "변경 후 24시간 내 취소 가능"
- 하단 CTA: "KB국민카드로 바꾸기" → Screen 10

**Screen 10 [TO-BE] — 변경 완료·후속 안내**
- 성공 아이콘 + "변경이 완료되었어요"
- 요약 카드: "다음 자동납부부터 KB국민카드로 출금됩니다"
- **다음 할 일** 리스트 카드:
  - "다음 달 예상 금액 확인하기 →" (Screen 5 연결)
  - "가족 회선도 같이 관리하기 →"
- 하단 CTA: "요금 허브로 돌아가기" → Screen 2

## §4 전역 인터랙션 / 상태

- 모든 금액 변동은 숫자 카운트업 애니메이션(약 300ms)
- 스텝 간 전환은 좌→우 슬라이드(약 200~250ms)
- 바텀시트는 상단 grabber, 배경 dim
- 로딩은 스켈레톤 우선(스피너 지양)
- 성공 피드백은 가벼운 마이크로 모션(아이콘 scale 1→1.05→1)
- 오류 상태는 해당 필드/카드만 강조, 전체 화면 톤은 유지(패닉 유도 금지)
