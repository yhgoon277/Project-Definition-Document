## 과제별 핵심 요약

### 1. AI 기반 탐색·추천 경험 설계
- **목적**: AI Agent 원스톱 경험 / 컨시어지형 추천 / 데이터 트래킹 체계
- **문제**: 탐색–의사결정–업무처리 분리, 개인화 부재, 데이터 분산
- **Pain Point**: 업무 흐름 단절, 맞춤정보 부재, 챗봇 신뢰도 저하
- **핵심기능**: AI Agent 진입, 비교/의사결정 지원, Next Action 오퍼링, 행동 트래킹

### 2. AI가 완성하는 상품 선택 설계
- **목적**: 상품·혜택·할인 데이터 표준화 → 이해·비교·시뮬레이션·선택 끊김 없는 경험
- **문제**: 상품/할인 정보 분산, 단계별 분절, 정확한 정보 제공 곤란
- **Pain Point**: 상품 구조 복잡성, 비교/시뮬레이션/추천 미연결
- **핵심기능**: 상품 등록/관리, 비교/시뮬레이션, AI 추천안 생성, 설득 논리 제공

### 3. 계약 및 가입 정보 관리 체계 기획
- **목적**: MY 통합 대시보드 / 고객 중심 통합 제공 / 셀프 완결
- **문제**: T4s 통합 시 정보량 과다, 고정된 메뉴 구조, 액션 탐색 피로
- **Pain Point**: 회선단위 통합 관리 곤란, 분산된 기능, 채널 이동 필요
- **핵심기능**: 통합 가입정보 확인, 해지/정지/분실 통합 진입, 긴급 업무 허브

### 4. 신뢰 기반 셀프완결형 요금 납부·관리 경험 재설계
- **목적**: 의미 이해 중심 구조 전환 / 사전 예측·관리 / 셀프완결형 처리 / 가족·다회선 확장
- **문제**: 기능 분산, 청구 금액 중심, 단절된 여정, 예외 상황 대응 한계
- **Pain Point**: 금액 이해 곤란, 예측 어려움, 미납·자동납부 실패 해결 곤란
- **핵심기능**: 요금/납부 통합 허브, 미납 상태 확인, 자동납부 정책 관리

### 5. 결제 효율화 체계 구축
- **목적**: 복잡한 금액/청구 구조를 직관적으로 이해하고 결제까지 완료
- **문제**: 결제 금액·수단·고객정보가 상품/채널별로 상이하게 운영
- **Pain Point**: 복잡한 금액 구성, 제한적 결제수단, 셀프처리 완결성 저하
- **핵심기능**: 결제 요약, 할인 적용(포인트·쿠폰·제휴카드), 결제수단 선택, 결제 실행/인증

### 6. 셀프 주도형 해결 중심 고객지원 통합 체계 구축
- **목적**: 분산된 고객지원 기능 통합 / 문제 유형 중심 전환 / 해결 중심 흐름 / 온·오프라인 맥락 연결
- **문제**: 기능 분산, 연결 구조 미흡, 챗봇 보조 역할, 셀프 처리 중단
- **Pain Point**: 해결 위치 판단 부담, 실행 연결 부족, 챗봇 신뢰도 저하
- **핵심기능**: 고객지원 허브 통합 진입, 문제 유형 중심 라우팅, 해결-상담 연결 흐름

### 7. 주문 사후관리 체계 고도화
- **목적**: 교환/반품/환불/해지 등 주문 사후 업무 통합 관리
- **문제**: 사후관리 기능이 분산되어 고객이 직관적으로 이용 불가
- **Pain Point**: 가용 액션 파악 어려움, 셀프서비스 불가 → CS 의존
- **핵심기능**: 주문 이력 조회, 배송 관리/변경, 클레임 처리(교환/반품/환불/해지), 상품 통합/개통

### 8. 고객 주문경험 혁신 프로세스 재설계
- **목적**: 표준화된 공통 주문 프로세스, 모듈형·재사용 가능 설계
- **문제**: 상품/유형별 다른 주문 단계, 앱/웹 이원화, 비대면 완결률 저조
- **Pain Point**: 복잡한 단계, 반복 입력, 채널별 다른 흐름
- **핵심기능**: 로그인 상태 정의, 상품 선택/구성 모듈, 주문 정보 입력, 약관/인증, 결제, 상태 관리, 개통, 온보딩

### 9. 전시 구조 및 노출 정책 기획·운영
- **목적**: 채널 전시 구조를 Atomic 단위로 재설계, 맥락·세그먼트 기반 차별 구성
- **문제**: T4S 전시가 채널별 분산, 링크/메뉴 중심 구조, 서비스 간 흐름 단절
- **Pain Point**: 무관한 정보 반복, 자주 쓰는 기능 미노출, 탐색 부담
- **핵심기능**: Atomic 전시 요소 구조, 맥락 기반 차별 구성, 개인화 대응 구조

### 10. 이벤트·멤버십 혜택 및 리워드 통합 관리 체계 기획
- **목적**: 분산된 혜택 발견-사용 프로세스 통합, 재방문 유도
- **문제**: 혜택 발견-사용 분산, 채널별 이벤트/리워드 개별 관리, 일회성 이벤트 중심
- **Pain Point**: 혜택 인지 어려움, 소모적 혜택 배포, 운영 자원 분산
- **핵심기능**: 재방문 리마인더, 마감/리워드/혜택/참여 알림, 통합 이벤트/혜택 허브

### 11. 예측형 지원 기반 상품·서비스 이용 편의성 강화
- **목적**: 구독 상품 자산의 통합 관리, 예측형 지원
- **문제**: 채널별 다른 구조/방식, 높은 복잡도
- **Pain Point**: 채널 간 기능 격차, 기능 나열형 구조, 자산 맥락/안내 부재
- **핵심기능**: 데이터 관리, 충전/선물 자동화, 예외 처리, 통합 쿠폰/바우처 뷰, 미사용 자산 케어

### 12. 통합회원정보 기반 계정·인증·알림·약관 관리체계 구축
- **목적**: 채널 간 분산된 계정/인증/알림/약관을 통합 관리
- **문제**: T4s 채널별 분산, 인증 방식 불일치, 개별 채널 관리
- **Pain Point**: 일관된 관리 어려움, 인증 불편, 알림 정보 유실/과부하
- **핵심기능**: CI 기반 통합 계정, SSO/PASS/소셜 로그인, 인증 모듈 공통화, 통합 알림 센터, 통합 약관 관리

## 교차 분석: 공통 전략 테마

### 5대 공통 가치
1. **셀프완결 경험 강화** — 콜센터·유통망 의존 감소 (전 과제 공통)
2. **AI 기반 개인화** — 고객 맥락 반영 맞춤 경험 (#1, #2, #6, #9, #11)
3. **원스톱 통합** — 분산된 기능의 통합 (전 과제 공통)
4. **신뢰 기반 설계** — 고객 언어·이해도 중심 (#4, #5, #6)
5. **데이터 기반 최적화** — 행동 트래킹·지속 개선 (#1, #2, #9)

### 과제 간 의존 관계
```
[#12 통합회원/계정] ──── 기반 인프라 (전 과제의 전제)
[#9 전시 구조] ────── UX 기반 (전시/노출의 전제)
[#8 주문 프로세스] → [#7 주문 사후관리] (순차적)
[#2 상품 선택] → [#5 결제] → [#4 요금 관리] (구매 여정)
[#1 AI 탐색/추천] ── 전 영역에 걸쳐 적용
[#10 이벤트/멤버십] ── 전 영역 인게이지먼트 보완

```
