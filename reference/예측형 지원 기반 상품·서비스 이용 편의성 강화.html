과제 개요

고객 여정채널 운영/경험
업무 영역서비스 및 상품 이용 경험 기획/관리
과제 목적

가입 상품·서비스에서 발생하는 자산의 이용 구조를 재설계하여, 고객의 통신 자산 이용 편의성과 혜택 체감도를 제고하고 자산 활용 효율을 극대화하기 위함

문제 정의 요약현 상태

가입상품에서 발생하는 다양한 자산이 채널별·유형별로 서로 다른 구조와 이용 방식으로 제공되어, 고객이 이해하고 활용하기에 복잡도가 높음

Pain Point
  • 자산별·유형별 특화 기능 구현/ 최적화로 채널 간 기능 격차·불균형 발생
  • 고빈도 업무 중심이 아닌 기능 나열형 구조로 인해 핵심 시나리오 기준의 프로세스 효율화 미흡
  • 자산의 속성, 사용 맥락, 공유 맥락을 반영한 정보 구조 및 정보 안내 부족
Root Cause

자산 관리 과정 전반에서 접근 복잡성, 이용 방식의 이질성, 정보 전달 부족, 시스템 제약

미해결 시 Risk
  • 통합 채널 내 자산 관리 복잡도 지속에 따른 고객 혼선 심화
  • 혜택 인지 및 이용률 저하로 인한 상품 만족도 하락
  • 자주 사용하는 핵심 업무의 이용 불편 지속에 따른 서비스 사용성 저하
  • 이용·충전·공유 전환 저하에 따른 자산 활용도 감소


과제 정의

고객이 가입한 상품·서비스를 실제로 잘 이용할 수 있도록,
요금제/부가서비스/구독상품/로밍 등 가입 상품·서비스에서 발생하는 자산(데이터, 통화, 쿠폰, 할인바코드, 이용권 등)의 이용 구조 및 경험을 통합적으로 재설계한다.


As Is 문제점

번호문제점 (서비스의 문제점)영향 (고객에게 미치는 영향)출처
1

데이터, 통화, 쿠폰, 할인바코드, 이용권 등 정보가 분산되어 있다

  • 고객이 자신의 자산을 이용하려면 여러 메뉴와 서비스 영역을 오가야 한다

T4S분석

2

자산별 특화 기능이 구현되어 있으며, 고객에게 제공하는 정보의 수준/ 표현/ 순서/ UIUX가 모두 다르다

  • 고객이 자산 유형별로 정보를 매번 새롭게 해석해야 하며, 이 과정에서 learning curve가 발생한다

T4S분석

3

사용 경로가 직관적이지 않고 조회 중심 구조에 머물러 있다

  • 고객은 정보 조회 이후 필요한 업무를 앱에서 바로 처리하지 못하고, 다시 탐색해야 한다
  • 결국 셀프 업무처리 실패 후 상담으로 이탈한다

T4S분석

4

데이터 관련 정책이 너무 복잡하고 한눈에 보이지 않는다

  • 고객은 여러 데이터 선물/충전 방식의 차이·한도·횟수를 이해하기 어렵다 (T끼리 데이터 선물, 리필쿠폰 선물/리필, T가족모아 데이터 공유, 선불쿠폰 구매/충전, 팅요금제 충전선물, 데이터 한도 요금제 충전, 팅/ZEM플랜/안심음성 요금 충전, PPS 데이터 충전...)
  • 고객이 선물/충전 가능 여부를 시도 후에야 알게 되고, 실패 경험이 누적된다

고객조사

5

대용량 데이터 선물, 다수 수신자 동시 처리 같은 업무가 시스템적으로 불가능하다

  • 고객은 한 명씩, 1GB씩 반복 처리하며 비효율을 경험한다
  • 처리 시간과 조작 피로가 커져 앱보다 전화가 더 빠르다는 인식이 강화된다

VoC분석

6

제공되는 혜택은 다양하지만, 고객이 '오늘·지금 즉시 이용할 것' 중심으로 재구성되어 있지 않다

  • 고객이 이용할 혜택을 직접 찾고 해석해야 한다
  • 방문 트리거가 약해지고, 혜택이 있음에도 실제 이용 전환이 낮아진다

고객조사

7

혜택 사용을 적극적으로 유도·케어하는 체계가 부족하다

  • 고객이 혜택을 사용하지 못한 채 놓치는 경우가 발생하며, 누적되면 상품·서비스 체감 가치 저하로 유지율이 하락한다

T4S분석

8

가족 단위 고객의 실사용 시나리오를 앱이 충분히 받쳐주지 못한다

  • 고객은 자녀·배우자·부모 회선의 구독, 데이터, 혜택, 쿠폰을 대신 관리하기 어렵다
  • 전화/ 오프라인 의존이 높아지고, 보호자 고객의 불만이 커진다

VoC분석

9

공휴일·주말·야간에는 해지, 변경, 신청, 충전, 인증 등 MY 영역의 핵심 업무가 제한되는 경우가 잦다

  • 고객은 필요도가 가장 높은 시점에 앱이 무력하다고 느끼며 채널 신뢰도를 잃게 된다

기대 효과

측정 지표 (KPI)

나의 자산을 한눈에 이해하는 통합 관리 구조로 전환해야 한다

  • 데이터, 통화, 쿠폰, 할인바코드, 이용권을 ‘내 자산’ 기준 단일 화면으로 통합 구성한다
  • 자산별로 사용 가능 상태/ 만료 예정/ 실행 가능 액션을 보여준다
  • 고객이 여러 메뉴를 오가지 않고도 현재 보유한 자산 상태를 빠르게 파악할 수 있다
  • 탐색 시간과 메뉴 이탈이 줄고, MY 영역의 진입 가치가 높아진다
  •  자산 조회/ 사용 관련 FAQ 및 상담 유입률

자산 유형이 달라도 같은 원리로 이해되는 공통 관리 모델을 만들어야 한다

  • 데이터, 통화, 쿠폰, 할인바코드, 이용권에 공통으로 적용되는 정보 구조 표준을 정의한다
    • 예: 보유 현황/ 사용 조건/ 만료일/ 사용 방법/ 다음 행동
  • 자산별로 상이한 용어를 고객 언어 기준으로 통일한다
  • 상세 화면, 리스트 화면, 사용/변경 화면의 UI 패턴을 공통화한다
  • 고객이 자산 유형별로 새롭게 학습하지 않아도 된다
  • 자산에 대한 이해 속도가 빨라지고, 기능 간 전환 부담이 줄어든다

조회 화면에서 그치지 않고 빠른 실행과 완료까지 이어져야 한다

  • 모든 자산 정보에 즉시 실행 CTA를 기본 탑재한다
    • 예: 사용하기/ 선물하기/ 리필하기/ 변경하기/ 등록하기
  • 처리 완료 후 적용 결과/ 남은 액션/ 연관 업무까지 이어서 안내한다
  • 조회 이후 재탐색이나 상담으로 빠지는 비율이 줄어든다
  • 셀프 처리율이 높아지고, MY가 실질적인 관리 허브로 기능하게 된다
  • 업무별 셀프 처리 완료율
  • 업무별 평균 소요 시간

데이터 관련 정책을 직관적으로 이해 가능하고, 즉시 실행 가능한 구조로 바꿔야 한다

  • 데이터 관련 정책을 슬림화한다
  • 다양한 데이터 선물/충전 수단의 차이를 한 화면에서 비교할 수 있게 구성한다
  • 고객의 조건을 사전 체크해서 실행 가능한 수단만 노출한다 
  • 정책 문구를 약관형 표현이 아니라 고객 지향적 표현으로 바꾼다
  • 고객이 실패 후 이해하는 구조에서, 실행 전에 이해하는 구조로 바뀐다
  • 데이터 관련 실패 경험과 문의가 줄고, 실행 전환율이 높아진다
  • 데이터 정책/ 조건 관련 FAQ 및 상담 유입률

시스템 제약을 없애 반복 업무를 한 번에 끝내는 일괄 처리 구조가 필요하다

  • 다수 수신자 동시 처리, 대용량 발송을 시스템적으로 가능하게 한다
  • 가족 회선, 자주 선물하는 대상, 최근 사용 이력을 기반으로 일괄 실행 UX를 구성한다
  • 반복 조작 시간이 크게 줄고, 앱이 전화보다 빠르다는 인식이 형성된다
  • 가족 단위 고객과 헤비 유저의 사용 만족도가 높아진다
  • 일괄 처리 관련 상담 유입률

전체 자산을 나열하는 대신 ‘지금 바로 쓸 수 있는 것’ 중심으로 노출되어야 한다

  • 만료 예정 혜택, 미사용 혜택, 놓친 혜택을 중심으로 이용 화면을 재편한다
  • 사용 가능한 자산에는 바로 실행 가능한 버튼을 전면 배치한다
  • 고객이 사용 가능한 자산을 찾는 시간이 줄고, 실제 사용 전환이 높아진다
  • 혜택 체감도

고객이 혜택을 찾아가지 않고 혜택이 고객을 찾아가야 한다

  • 혜택 알림을 일괄 발송이 아니라 상황 기반 이벤트 알림으로 전환한다.
    • 예: 만료 임박, 근처 사용처 진입, 시간대, 최근 행동, 이번 달 미사용 혜택 존재, 방금 사용 가능한 조건 충족
  • 알림 메시지에는 혜택명만 노출하지 말고 왜 지금 사용해야 하는지를 함께 전달한다
  • 고객은  적절한 순간에 사용 기회를 인지할 수 있다
  • 혜택을 못 써서 생기는 손실감과 불만이 줄고, 체감 가치가 올라간다
  • 혜택 알림 클릭률
  • 혜택 사용률

고객 seg.별 목적과 상황에 맞는 맞춤형 화면을 제공해야 한다

  • 고객 seg.별로 우선 노출 자산, 추천 액션, 퀵메뉴 구성을 다르게 제공한다
    • 고객 seg.는 요금제 유형, 보유 자산, 이용 패턴, 가족 여부, 미성년 여부 등을 기반으로 세분화한다
  • 고객이 자기와 관련 없는 정보를 덜 보게 되어 피로도가 낮아진다
  • 각 세그먼트에 맞는 실행 유도가 가능해져 전환율과 만족도가 함께 높아진다
  •  세그먼트별 주요 업무 완료율

주말·야간에도 끊기지 않는 24시간 셀프 처리 구조가 필요하다

  • 해지, 변경, 신청, 충전, 인증 등 핵심 업무를 운영시간 의존 없이 처리 가능한 구조로 재설계한다
  • 실시간 처리가 어려울 경우 즉시 접수/ 처리 예정 시간 안내/ 결과 알림 구조를 제공한다
  • 고객이 가장 급한 순간에 앱에서 문제를 해결할 수 있어 채널 신뢰도가 높아진다
  • 공휴일/ 야간/ 주말 셀프 처리 완료율
  • 운영시간 외 상담 유입률
  • 시간대별 업무 실패율
