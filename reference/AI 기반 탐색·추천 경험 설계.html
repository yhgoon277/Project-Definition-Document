과제 개요

고객 여정

채널 운영/경험

업무 영역AI 기반 탐색·추천 경험 설계
과제 목적
  1. 고객이 원하는 정보 탐색부터 의사결정, 업무처리까지 한 번에 수행할 수 있는 AI Agent 원스톱 경험 제공
  2. 고객의 상태·이력을 반영한 최적의 정보를 제시하고, Next Action을 제안하는 컨시어지형 추천 경험 제공
  3. 고객 행동 데이터 수집·트래킹 체계를 수립하고 서비스 고도화에 활용
문제 정의 요약현 상태
  • 탐색–의사결정–업무처리가 분리된 구조로, 검색은 정보 조회 중심, 비교·판단은 사용자 직접 수행, 가입·상담 등 업무 처리는 별도 메뉴에서 수행
  • 개인화 없이 획일적인 상품/혜택 전시 구조
  • 채널별 데이터 수집·트래킹 분산으로 통합 분석/활용 어려움
Pain Point
  • 탐색–의사결정–업무처리간 여러 경로를 반복 이동하며 업무 흐름 단절 (End-to-End 경험 부재)
  • 개인화 부재로 사용자 맞춤 정보·상품 탐색 부담 증가
  • Next Action 오퍼링 부재로 고객 행동 유도 기회 미활용
  • 비교/분석 및 의사결정 지원 미흡으로 선택 피로도 증가
  • 챗봇 및 추천 신뢰도 낮음 → 재탐색 및 상담 전환 발생
Root Cause
  • 통합 AI Agent 부재로 원스톱 실행 흐름 미구축
  • 개인화 기반 전시·추천 체계 미흡
  • 채널별 데이터 수집·트래킹 분산으로 통합 분석 기반 부족
미해결 시 Risk
  • 탐색–이탈–재탐색 반복 구조 지속으로 사용자 이용 피로도 증가
  • 개인화 미흡으로 추천 효율 저하 → 전환 기여도 미흡
  • 고객 행동 유도 실패 → 전환 기회 상실 및 수익화 저해
  • 데이터 기반 개선 및 최적화 불가 → 서비스 고도화 한계
  • AI Agent·개인화 경쟁력 확보 실패 → 경쟁 서비스 대비 열위

과제 정의

  • 고객이 원하는 정보 탐색부터 의사결정(비교·판단), 업무처리(가입·변경·상담 등)까지 원스톱으로 지원하는 AI Agent를 구축하고, 기존의 메뉴 탐색 및 챗봇을 대체하는 진입점을 제공한다.
  • 고객의 상태·이력·맥락을 반영해 필요한 정보와 행동을 전시·추천하는 컨시어지형 추천 체계를 구축한다.  
  • 고객 행동 데이터 수집·트래킹 체계 설계를 통해 AI 중심 서비스로 고도화할 수 있는 기반을 마련한다.

1

정보 탐색부터 의사결정, 업무처리·문제해결까지 "End-to-End 원스톱 Agent" 경험 제공

  1. Agent 정의
    • Agent 역할 정의 (예 : 정보 안내, 비교/분석, 추천, 업무처리)
    • Agent 페르소나 정의 (성격, 어투, 응답 스탠스  등)
    • 응답 기본 원칙 정의 (예 : 간결, 행동 유도형, 근거 제시형 등)
  2. Agent 기능 범위 정의
    • 각 기능별 Task 범위 정의
      • 예 : 정보 조회 → 데이터, 혜택 사용이력 등 조회할 정보 범위
      • 예 : 업무 처리 → 가입, 변경, 해지 등 지원할 업무 처리 범위
  3. Agent 에 필요한 데이터 학습
    • 학습 데이터 범위 선별 및 학습  (예 : 서비스 IA, 상품/서비스 정책서, 고객 데이터 등)
  4. Agent 각 Task 별 대화 Flow 설계 
    1. Task에 맵핑되는 자연어 기반 intent 설계
      • intent별 핵심 발화 패턴 정의 (동의어/변형/축약 등 포함)
      • Intent별 필수,옵션 파라미터 정의
    2. 응답 시나리오 설계
      • intent 별 자연어 응답 시나리오 정의 
      • 시나리오별 응답 UI 템플릿 정의 
      • 사용자 입력 부족시 보완 질문 및 단계별 인터랙션 방식 정의
      • 대화 예외 처리/실패 시나리오 정의
  • 탐색-의사결정-업무처리가 단일 흐름으로 통합된 Seamless Experience 구현
  • 사용자 이동/이탈 최소화
  • 고객 행동을 결과(처리 완료)까지 견인하는 실행 중심 경험 제공
  • Agent 진입 후 업무 검색→실행 전환율 (End-to-End)
  • Agent 업무 완결율 
  • Agent 셀프 처리율
2

비교의 수고는 줄이고 선택의 확신은 높이는, "AI 기반 의사결정 케어"

AI 기반 비교/분석/추천 시스템 구축

  1. 대상 정의 (요금제 / 상품 / 혜택 등)
  2. 기준 정의 (가격, 혜택, 조건 등 핵심 요소 구조화)
  3. 데이터 구조 정리 (상품 간 비교 가능하도록 정형화)
  4. 사용자 상황 기반 추천 로직 정의 (사용량, 이력 등 반영)
  5. 추천 결과 및 근거 표현 방식 정의 (이유, 차이점 등)
  6. 의사결정 UI 구성 정의 (비교표, 추천 강조, 선택 유도 요소)
  • 사용자 의사결정 시간 단축. 선택 피로도 제거
  • 비교·추천 기반으로 ‘고민 없는 선택’ 경험 제공
  • 고객 상황에 최적화된 선택지 제시로 의사결정 품질 향상
  • 실행률 상승
  • AI 기반 비교·추천의 가입/구매 전환율

3

내 마음을 읽어주는 기술, 나만의 정답을 찾아주는 "AI 추천"

  1. 추천 항목 정의 (예 : 요금제, 단말, 구독상품, 부가서비스 등)
  2. 필요한 데이터 항목 정의
  3. 추천 기준 및 로직 설계
  4. 추천 결과 노출 위치 및 방식 정의
  5. 추천 성과 측정 트래킹 지표 정의 (전환율, 클릭율, 수용율 등)
  • 서비스 접점별(예:상품목록,배너 등) 추천 로직이 일관된 정책으로 운영
  • 추천 신뢰도 향상
  • 전환 기여 확대
  • 추천 신뢰도 및 전환율 향상
  • 추천 이후 가입/주문 전환율
4

고객 니즈에 사업적 관점을 더하는, 수익성 가드레일 기반 AI 추천 체계

  1. 수익성 가드레일 정책 수립
    (마진, 비용, 프로모션 한도, 우선 노출 기준 등 사업 제약 조건 정의)
  2. 추천 로직 내 고객 가치 + 사업 가치 통합 설계
    (추천 스코어링 시 고객 적합도와 수익성 지표를 동시에 반영)
  3. 가드레일 기반 추천 결과 통제 및 운영 기준 정의
    (비수익/과도한 비용 유발 추천 제한, 예외 처리 정책 수립)
  4. 추천 성과 및 수익 기여도 측정 → 지속 최적화 체계 구축
    (전환율, ARPU, 비용 대비 효과 등 KPI 기반 개선 루프 운영)
  • 고객 니즈와 수익성을 동시에 반영한 추천으로 전환율 및 매출 기여도 향상
  • 추천 기반 전환율
  • 추천 기반 매출 기여도
5

성과를 스스로 탐지하고, 개선하는 "AI 셀프진화형 추천 체계"

  1. 추천 시나리오 성과 모니터링 및 이상 탐지 기준 정의
    (전환율 저하, 클릭률 감소 등 자동 감지 기준 수립)
  2. AI 기반 개선 제안 생성 구조 설계
    (추천 로직/대상/노출 방식 등에 대한 개선안 자동 도출)
  3. AI 제안에 대한 운영자 검토 및 승인/반려 프로세스 정의
    (추천 시나리오 변경 전 검증 및 통제 구조 마련)
  4. 승인된 제안의 즉시 반영 및 배포 구조 설계
    (추천 정책/시나리오 실시간 업데이트 가능 구조)
  5. 개선 결과에 대한 성과 측정 및 반복 학습 구조 구축
    (A/B 테스트 및 성과 피드백 기반 지속 최적화)
  • AI가 성과 트래킹 & 개선방향 제안 → 사람이 accept → AI가 개선방향 반영
  • 추천 시나리오 기획 리소스 절감
  • 신속한 정책 반영
  • 추천 전환율 개선폭
  • 추천 시나리오 기획 리소스 절감률
  • 정책 반영 리드 타임
6

고객의 필요가 시작되는 순간, 고객의 니즈를 선점하는 "Next Action 오퍼링"

  1. Next Action 행동 유형 정의 (예 : 가입, 변경, 혜택 사용 등)
  2. Next Action 오퍼링 트리거 조건 설계
  3. 상황별 추천 Next Action 로직 설계
  4. 로직간 우선순위, 예외처리 정의
  5. Action 제안 UI/노출 방식 정의 (배너, 카드, CTA 등)
  • Next 행동 유도 강화
  • 전환 기회 확대
  • 서비스 이용률과 수익 기회 확대
  • 추천된 Next Action 실행율
  • 업무 End-to-End 완료율
7

고객의 모든 순간을 가치로 잇는, 전방위적 "데이터 수집·트래킹 체계"

  1. 사용자 행동 이벤트 정의 (클릭, 노출, 스크롤, 이탈 등)
  2. 이벤트 측정 구간 및 기준 정의 (세션/화면/Flow 단위 등 설계)
  3. 이벤트 발생 지점 정의 (어느 화면/컴포넌트/Action에 트래킹 삽입할지)
  4. 이벤트 파라미터 구조 정의 (user, 상품, 맥락 정보 등 스키마 설계)
  5. 성과 분석/리포팅 체계 정의 (퍼널, 전환, 이탈 분석 구조)
  • 데이터 기반 의사결정 가능
  • 지속적, 통합적인 서비스 개선
  • 트래킹 커버리지 (전체 고객행동 중 트래킹 비율)
  • 트래킹을 통해 파악가능한 지표/인사이트
8

AI의 정교함과 운영의 유연함을 더하는, 빈틈없는 "AI 운영 체계"(admin)

AI 결과에 대한 운영자 보정/제어 구조 Admin 설계

  1. AI 추천/전시 정책 설정 기능 설계
    • 대상 설정 (예 : 5G요금제 사용자, 최근 7일내 로그인 사용자)
    • 전시 항목 (예 : 특정 요금제, 특정 혜택, 특정 Action(가입 유도) 등)
