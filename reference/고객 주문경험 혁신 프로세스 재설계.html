문제 정의 요약현 상태상품/유형별 주문 단계와 인증 방식이 상이하고, 앱/웹 이원화 및 기보유 정보 재입력 구조로 인해 주문 절차가 복잡하게 운영되고 있음.
Pain Point복잡한 단계와 반복 입력, 채널별 상이한 흐름으로 고객 피로와 혼선이 발생하고, 셀프개통 불편으로 비대면 처리 완결성이 낮음.
Root Cause공통 주문 프로세스와 정책 표준화 미흡, 앱/웹 분리 운영, 고객정보 재활용 및 인증 통합 체계 부족이 주요 원인임.
미해결 시 Risk주문 이탈 및 고객 불만 증가, 셀프처리율 정체, 오프라인 의존 지속, 운영비용 및 정책 대응 비효율이 심화될 수 있음.

측정 지표 (KPI)

1. 주문 프로세스는 상품별 화면 묶음이 아니라 재조합 가능한 "표준 모듈 구조"로 설계되어야 한다

  • 가입, 본인인증, 명의 검증, 가입 가능 조건 체크, 약관 동의, 결제, 개통, 후속 연결 등 유사 단계를 공통 주문 모듈로 분리한다.
  • 상품별 주문 프로세스는 공통 모듈 조합 방식으로 구성하여 신규 유형의 상품이 출시되더라도 빠르게 적용 가능하도록 설계 원칙을 수립한다.
  • 예외 케이스도 개별 화면 추가 방식이 아니라 표준 모듈 내 분기 규칙으로 관리한다.
  • 현재 상품 간 프로세스 중복과 운영 복잡도를 줄일 수 있다.
  • 향후 신규 상품 추가 시에도 빠르게 적용 가능해진다.

2. 표준화된 주문 구조는 "AI 기반 주문 지원"을 가능하게 하는 기반이 되어야 한다

  • 상품·유형마다 다른 인증, 처리 방식을 줄이고 주문 단계, 상태, 예외, 실패 유형이 구조화한다. 
  • 주문 단계별 상태값, 실패 유형, 보류 사유, 고객 조건, 필요한 입력값을 표준 스키마로 정의한다.
  • AI가 각 단계의 맥락을 이해할 수 있도록 주문 프로세스별 의도, 슬롯, 분기 조건, 다음 액션 후보를 정리한다.
  • FAQ 수준이 아니라 실제 주문 맥락에서 다음 행동을 제안할 수 있도록 프로세스 데이터와 고객 상황 데이터를 연결한다.
  • 고객에게는 일관된 주문 경험을 제공하고, 내부적으로 운영 효율 및 향후 AI 주문 도우미와 자동 지원 기능을 제공하게 된다.
  • AI가 단순 답변이 아니라 실제 주문 진행을 도와주는 구조로 진화할 수 있다.
  • 향후 주문/가입 프로세스에 AI를 붙이더라도 별도 우회 설계 없이 자연스럽게 확장 가능해진다.

3. 가입/주문 과정은 ‘길이’보다 ‘생각할 양’과 ‘입력할 양’을 줄이는 방향으로 재설계되어야 한다

  • 한 화면에서 동시에 비교·판단해야 하는 정보량을 줄이고, 단계별로 필요한 결정만 하도록 분해한다.
  • 가격, 혜택, 약정, 개통 방식은 고객이 헷갈리는 지점에서만 요약 비교형으로 노출한다.
  • 고객이 “쉽고 빠르게 주문했다”고 체감할 수 있게 된다. 
  • 가입 중 피로와 망설임이 줄어 주문 완료율이 올라간다.

4. 고객 정보는 "다시 입력하게 하지 않는 것"이 기본 원칙이 되어야 한다

  • 회사가 이미 보유한 고객 정보와 직전 단계 입력값, 인증 결과를 다음 단계에 자동 반영·재활용 한다.
  • 오류, 이탈, 앱 재진입 이후에도 이전 진행 상태를 복원해 이어서 주문할 수 있게 한다.
  • 고객은 “다시 입력해야 한다”는 피로 없이 쉽고 빠르게 주문했다는 체감을 얻게 된다.
  • 재입력으로 인한 포기와 전화 전환이 줄어든다.

5. 주문 품질은 오류 안내보다 "오류 발생 자체를 줄이는 구조"에서 시작되어야 한다

  • 유심 인증 실패, IMEI 불일치, eSIM 개통 오류, 주문 보류, 앱 접속 오류 등 반복 이슈를 유형별로 분류하고 우선순위를 부여한다.
  • 고객이 자주 막히는 프로세스를 사전 검증 로직, 입력값 점검, 예외 처리 자동화로 재설계한다.
  • 고객은 안내를 많이 받기 전에 애초에 덜 막히게 된다.
  • 셀프개통 완료율과 주문 완결률이 올라가고 오류성 상담이 감소한다.

6. 고객이 막히는 순간에는 원인, 영향, 다음 행동이 "즉시 이해"되어야 한다

  • 주문 가능 조건, 실패 원인, 보류 사유, 재시도 방법, 담당 부서 연결 등 해결 방법은 진행 중 바로 제공되어야 한다.
  • 오류 코드 기반 자가 해결 가이드와 원클릭 재시도, 후속 문의 연결을 함께 제공한다.
  • 고객이 “왜 안 되는지”와 “지금 무엇을 해야 하는지”를 즉시 이해할 수 있도록 고객 언어로 표현한다.
  • 고객은 “왜 안 되는지 모르겠다”에서 “어떻게 해결하면 되는지 안다”로 전환된다.
  • 실패가 곧 이탈로 이어지는 비율이 낮아진다.

7.  고객이 몰라서 주문을 못하는 상황을 없애기 위해 "AI 주문 도우미" 를 지원해야 한다

  • 텍스트 안내 중심에서 단계형 시각 안내 중심으로 전환하여 고객이 필요한 시점에 이해할 수 있도록 맥락형 안내를 제공한다.
  • 개통 단계별 진행 상태, 필요 준비물, 실패 시 복구, 상담 연결, 재진입 이어하기 기능을 기본 제공한다.
  • AI는 추천보다 먼저, 주문을 계속 진행할 수 있게 돕는 실시간 안내자 역할을 해야 한다.
  • 고객이 몰라서 불안해 하는 상황을 막고, 주문 중간 단계에서 멈추는 사례를 줄일 수 있다.

8. "셀프개통"은 주문의 부속 단계가 아니라 별도의 "핵심 완결 경험"으로 설계되어야 한다

  • 셀프개통은 단계형 UX, 오류 복구, 재진입 지원, 도움 연결까지 포함한 독립 여정으로 설계한다.
  • 주문 완료 직후 개통, 유심 활성화, 확정기변, 앱 연동, 이용권 발급, 계정 연동, 후속 설정을 하나의 후속 플로우로 제공한다.
  • “다음에 해야 할 일”을 주문 종류별로 자동 제시하고, 필요한 경우 즉시 실행 버튼으로 연결한다.
  • 고객은 주문 완료와 실제 사용 시작 사이의 단절을 느끼지 않게 된다.
  • 가입했지만 못 쓰는 상태가 줄고, 초기 만족도와 사용 전환율이 높아진다.

9. "주문의 끝은 신청 완료가 아니라 결과를 이해하고 실제 사용 시작까지 포함"해야 한다

  • 현재 요금제 대비 후보 요금제 실납부액, 할인 변화, 약정 영향, 연계 혜택 변화, 개통 이후 달라지는 이용 조건을 사전 비교형으로 제공한다.
  • 주문-배송-개통-혜택 적용 상태를 하나의 대시보드에서 조회할 수 있게 한다.
  • 주문 후 개통, 이용 시작, 연계 서비스 연결, 혜택·할인·이용조건 변화까지 하나의 흐름으로 안내한다.
  • 고객은 신청 후 결과를 확인하는 것이 아니라, 결과를 이해한 뒤 선택하게 된다.
  • 비용 불신과 혜택 관련 문의가 감소하고, 주문 신뢰도가 높아진다.
  • 주문에 대해 고객이 이해하고 바로 서비스를 사용할 수 있게 된다.

  • 가격/혜택/할인 정보 확인
  • 예상 납부 금액 시뮬레이션 일부 제공
  • 가입 조건의 일반 기준 안내
  • AI 주문 도우미 기반 상품 설명 및 비교 보조
  • 비로그인 고객도 상품상세 화면까지는 자연스럽게 진입 가능해야 한다.
  • 고객은 로그인 전에 먼저 “살 만한지”, “나한테 맞는지”를 보고 싶어 한다.
3상품 선택/구성 모듈

  • 2-1. 메인상품 선택

    • 단말 / 요금제 / 가입유형 / 개통유형 선택
    • 상품 간 비교표 제공
    • 핵심 차이점 요약
    • AI 상품 요약 정보 제공

    2-2. 추가상품 선택

    • 선택한 메인상품 유형에 따라 연계 가능한 추가상품 제안
    • 부가서비스, 보험, 구독, 액세서리, 연계 혜택 상품 제안
    • 필수 선택/선택 가능 항목 구분
    • 상품 유형별 종속 관계 반영

    2-3. 사은품 선택

    • 상품 유형별 제공 가능한 사은품 노출
    • 선택 가능한 사은품 비교
    • 사은품 제공 조건 안내
    • 지급 예정/조건/제외 사유 안내

    2-4. 비교/추천/시뮬레이션

    • 상품 비교 기준 표준화
    • 현재 대비 변화 비교
    • 실납부액 사전 시뮬레이션
    • 할인/가입 정보 기반 개인화 시뮬레이션
    • Seg별 추천 상품 노출
    • BSS CVM 기반 가입 가능 상품 리스트 사전 체크 후 노출
    • 고객 할인/가입 정보 기반 후보 상품 우선 정렬
    • AI 추천 사유 / AI 요약 설명 제공
  • 이 모듈은 단순한 상품 상세 화면이 아니라 주문 전 의사결정의 중심 모듈이다.
  • 비로그인 고객에게는 공통 비교/탐색 중심으로, 로그인 고객에게는 가입 가능 상품 필터링, 개인화 시뮬레이션, 추천 강화 구조로 운영
4주문 시도 및 로그인 전환 모듈
  • 주문(바로주문/장바구니) 버튼 클릭 시 로그인 요청
  • 로그인 필요 이유 안내
  • 간편 로그인 / 본인 확인 진입
  • 로그인 후 이전 상품/선택값 복원
  • 이탈 없이 로그인 후 이어서 진행
  • 로그인은 탐색을 막는 장벽이 아니라 주문 진행을 위한 자연스러운 전환 지점이어야 한다.
  • 로그인 전 선택한 상품, 옵션, 비교 상태는 유지되어야 한다.
  • 고객에게 “다시 처음부터” 느낌을 주면 안 된다.
5로그인 기반 자격/조건 확인 모듈
  • 회선/명의/연령/약정/가입 상태 기반 자격 판별
  • 요금제 가입 가능 여부 확인
  • 단말/자급제/eSIM 지원 여부 확인
  • 변경 가능 조건 확인
  • 추가 정보 입력 필요 대상 분기
    • 14세 미만
    • 외국인
    • 법정대리인 필요 고객
    • 가족/타인 명의 처리 고객
  • 불가 사유 및 대안 경로 안내
  • AI 주문 도우미 기반 조건 해석 지원
  • 기본 원칙은 로그인 정보만으로 최대한 자동 판별하는 것이다.
  • 다만 14세 미만, 외국인, 가족/대리 처리처럼 예외 케이스는 추가 인증/추가 입력 분기가 필요하다.
  • 여기서 검토가 필요한 쟁점은 말씀하신 대로 2가지입니다.
    1. 로그인만으로 본인 확인이 충분한 업무 범위
    2. 자동 세팅 정확도를 위해 인증을 먼저 추가로 받아야 하는 업무 범위
  • 단순 조건 확인이 아니라 로그인 정보 기반 사전 판별 + 예외 분기 설계가 핵심입니다.
6주문정보 입력
  • 회선 확인
  • 주문 대상 회선 선택
  • 배송지/연락처/청구지 입력
  • 로그인 정보 기반 자동 입력
  • 기존 입력값 자동 세팅
  • 포인트,쿠폰,제휴할인 등 추가 할인 등록
  • 단계 간 입력값 유지
  • 이탈 후 재진입 시 이어하기
  • 가족/미성년자/외국인 전용 추가 입력 분기
  • AI 입력 보조
    • 용어 설명
    • 누락값 안내
    • 입력 오류 사전 점검
  • 회선 확인 단계는 별도 기능이 아니라 입력 모듈의 핵심 하위 기능으로 넣는 것이 맞다.
  • 고객이 이미 알고 있는 정보, 회사가 이미 보유한 정보는 다시 입력하게 하면 안 된다.
  • 주문 경험에서 가장 중요한 것은 입력량을 줄이는 것이다.
7약관동의·인증
  • 약관 통합 동의
  • 본인 인증
  • 명의 확인
  • 법정대리인 인증
  • 가족/타인 동의 인증
  • 외국인 고객 추가 인증 분기
  • 인증 결과 재사용
  • 인증 실패 원인 안내 및 복구
  • 간편 인증 수단 확대
  • 이 모듈은 “약관동의 + 인증”을 함께 보는 게 맞습니다.
  • 다만 실제 상품 특성에 따라 약관 먼저 / 인증 먼저 순서는 일부 달라질 수 있습니다.
  • 핵심은 상품마다 방식이 달라도 고객이 체감하는 경험은 같은 표준 구조여야 한다는 점입니다.
8결제
  • 결제수단 선택
  • 최근 사용 결제수단 재사용
  • 결제 정보 사전 저장 기반 간소화
  • 월 납부액 / 총 납부 금액 / 할부 조건 확인
  • 할인/혜택 적용 결과 확인
  • 실납부액 최종 확인
  • 신청 완료 및 주문번호 발급
  • 결제는 단순 결제창이 아니라 마지막 확신 단계다.
  • 고객이 “무엇을 신청했고, 얼마를 내는지”를 납득하고 끝내야 한다.

  • 배송 상태 조회

  • 사전적격심사 지원
  • 보류 사유 안내
  • 해피콜 예정 시간 안내
  • 지연 원인 안내
  • 재시도 / 취소 / 담당 부서 연결
  • 주문 이어서 처리하기
  • 주문 이후 불안의 핵심은 “지금 어디까지 됐는지 모른다”는 점이다.
  • 상태조회는 부가 기능이 아니라 주문 완결 경험의 핵심이다.

  • 기기변경 후 유심 활성화
  • 확정기변 처리
  • 단계형 개통 가이드
  • 준비물 안내
  • 오류 코드 안내
  • 자가 복구 가이드
  • 재시도 / 이어하기
  • AI 개통 도우미
  • 도움 요청 / 상담 연결
  • 개통은 주문의 부속이 아니라 실질적인 완료 단계다.
  • 상품 특성에 따라 이 단계는 생략 가능하다.
    예: 요금제 변경 일부는 개통 모듈 없이 완료 가능
  • 반대로 단말 주문, 기기변경, eSIM, 자급제 개통은 이 단계가 핵심이다.

  • 혜택/할인/이용조건 변화 안내

  • AI 온보딩 가이드
